\section{Artifact Overview}

This artifact supports the CAIS 2026 paper ``Vista: Verifier-in-the-Loop Agentic RL for Semantic Program Synthesis in Quantum Computing.'' The artifact contains the training code for Vista, the staged OpenQASM verifier used during agentic reinforcement learning, generation scripts for Vista and baseline checkpoints, and offline evaluation scripts for reproducing the reported semantic metrics.

The main artifact components are:
\begin{itemize}
  \item \codepath{examples/train/quantum/}: GRPO training scripts for the 4B quantum circuit model.
  \item \codepath{verl_tool/servers/tools/quantum_cpu.py}: the quantum verifier service used during tool-augmented rollouts.
  \item \codepath{verl_tool/servers/tools/utils/quantum_reward_cal.py}: staged reward computation for syntax feasibility, behavior similarity, expectation-value objective quality, and utility-oriented optimization.
  \item \codepath{quantum-code-generation/code/generation/}: scripts for generating OpenQASM circuits from Vista and baseline models.
  \item \codepath{quantum-code-generation/code/evaluation/}: scripts for parsing generated QASM, simulating circuits, computing relative entropy, checking expectation values, and writing summary statistics.
  \item \codepath{quantum-code-generation/code/data_generation/src/algorithms/}: graph-generation source files and included pickle inputs for the supported graph optimization primitives.
  \item \codepath{artifact/scripts/run_quantum_experiment.sh}: reviewer wrapper for generation plus evaluation, or evaluation of an existing raw generation file.
  \item \codepath{artifact/scripts/draw_vista_figures.py} and \codepath{vista_draw/}: headless wrappers and plot-ready inputs for regenerating paper figures.
  \item \codepath{artifact/tables/} and \codepath{artifact/scripts/show_paper_tables.py}: extracted CSV values for Tables 1--3 and a dependency-free Markdown display script.
  \item Released checkpoints and data on Hugging Face: \codepath{Benyucong/rl_quantum_4b}, \codepath{Benyucong/sft_quantum_circuit_gen_4B}, and \codepath{Benyucong/graph-data-quantum-rl}.
\end{itemize}

\section{Hardware and Software Requirements}

The minimal artifact inspection path requires Python 3.10 or 3.11 and does not require a GPU. Recomputing evaluation metrics from raw generated circuits requires the Python packages listed in \codepath{quantum-code-generation/code/evaluation/requirements.txt}, including Qiskit 1.2.4, qiskit-aer 0.16.1, and qiskit-qasm3-import 0.5.1.

Regenerating model outputs requires a GPU-capable machine with enough memory for the selected 3B/4B model checkpoint. Full Vista retraining is substantially more expensive: the paper-scale configuration fine-tunes a 4B model with GRPO and FSDP on 128 AMD GPUs for approximately 48 hours. The artifact also includes smaller single-node Slurm wrappers intended for sanity checks or partial reproduction.

\section{Evaluation Workflow}

The quickest artifact check is:
\begin{verbatim}
cd /path/to/rl-quantum
python3 artifact/scripts/summarize_eval_outputs.py \
  quantum-code-generation/code/evaluation/out
python3 artifact/scripts/show_paper_tables.py all
\end{verbatim}
These commands parse the included precomputed summary files and print the extracted paper Tables 1--3 as Markdown.

For a fresh GPU-backed run followed by plots derived from the new evaluation output, evaluators can use:
\begin{verbatim}
bash artifact/scripts/run_quantum_experiment.sh \
  --n-samples 50 \
  --out-dir artifact_runs/reviewer_smoke
EVAL_DIR=artifact_runs/reviewer_smoke/evaluation
FIG_DIR=artifact_runs/reviewer_smoke/figures
python3 artifact/scripts/draw_vista_figures.py \
  --evaluation-dir "$EVAL_DIR" \
  --only box,relative_entropy \
  --output-dir "$FIG_DIR" \
  --strict
\end{verbatim}
The first command writes raw generations and \codepath{summary_stats_VISTA.json}/\codepath{summary_VISTA_raw_data.csv}. The second command converts that new evaluation CSV into the objective-gap and relative-entropy plot inputs and runs the existing figure scripts. The remaining paper figures can be redrawn from packaged plot-ready inputs with \codepath{draw_vista_figures.py --input-dir vista_draw}.

To regenerate all packaged paper plots from the included \codepath{vista_draw} plot-ready CSV/JSON inputs and logs, evaluators can run:
\begin{verbatim}
python3 artifact/scripts/draw_vista_figures.py \
  --input-dir vista_draw \
  --output-dir artifact_runs/paper_figures \
  --strict
\end{verbatim}
This single command runs the packaged plot tasks for objective gap, compilability, relative entropy, scalability, per-primitive breakdown, training dynamics, verifier efficiency, stage-cost breakdown, latency breakdown, hardware reward stability, and real-device quality--efficiency trade-off. Fresh \codepath{evaluate_samples.py} output directly supports only the objective-gap and relative-entropy plots unless the corresponding aggregate tables/logs are also regenerated.

To recompute metrics from raw generated outputs, evaluators should run:
\begin{verbatim}
cd quantum-code-generation/code/evaluation
python3 -m venv .venv-eval
source .venv-eval/bin/activate
pip install -r requirements.txt
GEN_JSON=../generation/out/<RAW_GENERATION_JSON>
python3 src/evaluate_samples.py \
  "$GEN_JSON" \
  out \
  <MODEL>
\end{verbatim}
The command writes \codepath{summary_stats_<MODEL>.json} and \codepath{summary_<MODEL>_raw_data.csv}. These outputs should be compared with the corresponding paper table or figure.

To regenerate model outputs, evaluators with GPU access can run:
\begin{verbatim}
cd quantum-code-generation/code/generation
python3 -m venv .venv-gen
source .venv-gen/bin/activate
pip install -r requirements.txt
python3 generate_samples.py \
  --uid cais26 \
  --model_path Benyucong/rl_quantum_4b \
  --dataset Benyucong/graph-data-quantum-rl \
  --n_samples 200
\end{verbatim}

\section{Notes for the AEC}

Full retraining is not expected for kick-the-tires because it requires a large GPU cluster. The intended reproducibility path is to use the released checkpoint and raw generation outputs, then rerun the deterministic evaluation scripts. Real-QPU experiments depend on external hardware availability and should be evaluated from archived hardware logs unless the AEC explicitly requests a live run.

The current artifact includes graph-generation pickle inputs, precomputed summary JSON/CSV files, the public RL checkpoint \codepath{Benyucong/rl_quantum_4b}, the public SFT checkpoint \codepath{Benyucong/sft_quantum_circuit_gen_4B}, and the public dataset \codepath{Benyucong/graph-data-quantum-rl}. For a low-friction Results Reproduced evaluation, the final archive should also include the raw generated-circuit JSON files used as inputs to \codepath{evaluate_samples.py}.

The submitted artifact URL is \url{https://github.com/benyucong/rl-quantum}. If a DOI-backed archive is added later, list that archive URL in HotCRP as well.
